## Template scenario (for the feature-walkthrough-gif project)

**Family office IC research room — agent fills the private-markets pacing column, every figure source-linked, nothing leaks**
- Scene: A single-family-office CIO opens a NodeRoom for the quarterly investment-committee pre-read. The shared sheet is a private-markets pacing model: existing commitments, NAVs, and a column for next-quarter pacing that needs filling against fresh manager data. The CIO types `/ask fill the Q3 pacing column from the manager updates`. The Room NodeAgent looks first, claims an affected-range lock on just the pacing cells (read-only to others, still readable as context), reads each cell's version, and writes — but because review mode is on, the writes arrive as proposals the CIO approves at the cell. Each written cell carries its value plus a source citation back to the manager-update artifact. The analyst's private working notes on a manager they're souring on stay redacted to '[private draft]' in the shared trace. When the agent is done, the CIO clicks the room summary: a bounded, read-only brief the principal can read without ever touching the raw model.
- Hidden pain: At this level the spreadsheet is never the problem — the chain of custody is. An IC pre-read with an unsourced pacing number, or a working note about a manager that leaks into a family-visible surface, is a trust failure, not a formatting one. The CIO's real fear is an AI quietly committing a wrong figure that nobody can trace, and private working judgment escaping the room.
- Feature shown: Affected-range lock + per-element CAS (no clobber); review mode (autoAllow off to resolveProposal) for approve-at-the-cell; evidence-bearing cells with source citation; private-draft redaction to [private draft] (evidencePolicy mixed_requires_redaction); per-room traces as decision receipts; summarizeRoomForPrivate for the principal brief.
- Cultural meaning: This is recognition, not persuasion: it animates how a family office actually works — provenance before action, discretion as the product, the principal seeing a clean decision and not the plumbing, and AI allowed to do real work only because every step stays reviewable and human-controlled. It matches the in-house-for-control posture UBS GFO 2025 documents and the governed-AI adoption RBC 2025 describes, without a single word about luxury.
